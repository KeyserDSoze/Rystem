# Rystem.RepositoryFramework.Api.Client

[![NuGet](https://img.shields.io/nuget/v/Rystem.RepositoryFramework.Api.Client)](https://www.nuget.org/packages/Rystem.RepositoryFramework.Api.Client)
[![NuGet Downloads](https://img.shields.io/nuget/dt/Rystem.RepositoryFramework.Api.Client)](https://www.nuget.org/packages/Rystem.RepositoryFramework.Api.Client)

.NET HTTP client for consuming APIs generated by `Rystem.RepositoryFramework.Api.Server`. Wires an `IRepository<T, TKey>`, `IQuery<T, TKey>`, or `ICommand<T, TKey>` to remote HTTP endpoints with full support for interceptors, JWT authentication, and Polly retry policies.

---

## Installation

```bash
dotnet add package Rystem.RepositoryFramework.Api.Client
```

---

## Quick Start

```csharp
builder.Services.AddRepository<Product, int>(repositoryBuilder =>
{
    repositoryBuilder.WithApiClient(apiBuilder =>
    {
        apiBuilder
            .WithHttpClient("https://api.example.com")
            .WithDefaultRetryPolicy();
    });
});

// Inject and use as usual
public class ProductService(IRepository<Product, int> repository)
{
    public async Task<Product?> GetAsync(int id)
        => await repository.GetAsync(id);
}
```

## DI Setup

The client is configured via `.WithApiClient(...)` on the repository builder. The builder is returned from `AddRepository`, `AddQuery`, or `AddCommand`.

### Full Repository

```csharp
builder.Services.AddRepository<Product, int>(repositoryBuilder =>
{
    repositoryBuilder.WithApiClient(apiBuilder =>
    {
        apiBuilder.WithHttpClient("https://api.example.com");
    });
});
```

### CQRS — Query only

```csharp
builder.Services.AddQuery<Product, int>(queryBuilder =>
{
    queryBuilder.WithApiClient(apiBuilder =>
    {
        apiBuilder.WithHttpClient("https://api.example.com");
    });
});
```

### CQRS — Command only

```csharp
builder.Services.AddCommand<Product, int>(commandBuilder =>
{
    commandBuilder.WithApiClient(apiBuilder =>
    {
        apiBuilder.WithHttpClient("https://api.example.com");
    });
});
```

## `IApiRepositoryBuilder<T, TKey>`

| Method | Description |
|---|---|
| `WithHttpClient(string domain)` | Creates an `HttpClient` with the given base address. Returns `HttpClientRepositoryBuilder`. |
| `WithHttpClient(Action<HttpClient>)` | Creates an `HttpClient` with custom configuration. Returns `HttpClientRepositoryBuilder`. |
| `WithStartingPath(string path)` | Sets the route prefix. Default: `"api"` |
| `WithVersion(string version)` | Adds a version segment to the URL (e.g. `"v1"`) |
| `WithName(string name)` | Overrides the route entity name segment (default: `typeof(T).Name`) |
| `WithServerFactoryName(string name)` | Appends a named-factory segment for multi-instance servers |

### URL structure

The final endpoint URL is built as:

```
{startingPath}/{version}/{Name}/{factoryName}/{method}
```

Examples with defaults (`startingPath = "api"`, `Name = "Product"`):

| Operation | URL |
|---|---|
| `GetAsync` | `api/Product/Get?key={key}` |
| `ExistAsync` | `api/Product/Exist?key={key}` |
| `DeleteAsync` | `api/Product/Delete?key={key}` |
| `InsertAsync` | `api/Product/Insert?key={key}` |
| `UpdateAsync` | `api/Product/Update?key={key}` |
| `QueryAsync` | `api/Product/Query/Stream` (streamed) |
| `BatchAsync` | `api/Product/Batch/Stream` (streamed) |
| `OperationAsync` | `api/Product/Operation?op={name}&returnType={type}` |
| `BootstrapAsync` | `api/Product/Bootstrap` |

> For complex/JSON-serializable keys the key is sent in the POST body instead of the query string.

## `HttpClientRepositoryBuilder<T, TKey>`

Returned by `WithHttpClient(...)`. Exposes the underlying `IHttpClientBuilder` for Polly and other middleware:

```csharp
apiBuilder
    .WithHttpClient("https://api.example.com")
    .WithDefaultRetryPolicy();       // built-in circuit breaker policy
```

### `WithDefaultRetryPolicy()`

Registers a Polly advanced circuit breaker:
- Breaks at 50% failure rate
- Sampling window: 10 seconds
- Minimum 10 requests before breaking
- Break duration: 15 seconds

### Custom Polly policy

```csharp
apiBuilder
    .WithHttpClient("https://api.example.com")
    .ClientBuilder
    .AddPolicyHandler(Policy<HttpResponseMessage>
        .Handle<HttpRequestException>()
        .OrTransientHttpError()
        .RetryAsync(3));
```

## Interceptors

Interceptors run before (request) or after (response) every HTTP call. All are registered on `IServiceCollection`.

### Request interceptors

Run before the request is sent. Use them to add headers, tokens, tracing, etc.

```csharp
// Global — applies to every repository client
services.AddApiClientInterceptor<MyGlobalInterceptor>();

// Specific to a model type
services.AddApiClientSpecificInterceptor<Product, MyProductInterceptor>();

// Specific to model + key type
services.AddApiClientSpecificInterceptor<Product, int, MyProductIntInterceptor>();
```

Implement `IRepositoryClientInterceptor`:

```csharp
public class MyGlobalInterceptor : IRepositoryClientInterceptor
{
    public async Task<HttpClient> EnrichAsync(HttpClient client, RepositoryMethods method)
    {
        client.DefaultRequestHeaders.Add("X-Correlation-Id", Guid.NewGuid().ToString());
        return client;
    }
}
```

For model-specific interceptors implement `IRepositoryClientInterceptor<T>` or `IRepositoryClientInterceptor<T, TKey>`:

```csharp
public class MyProductInterceptor : IRepositoryClientInterceptor<Product>
{
    public Task<HttpClient> EnrichAsync(HttpClient client, RepositoryMethods method)
    {
        // runs only for Product repository calls
        return Task.FromResult(client);
    }
}
```

### Response interceptors

Run after the response is received. Use them for 401 retry, logging, etc.

```csharp
// Global
services.AddApiClientResponseInterceptor<MyResponseInterceptor>();

// Specific to model
services.AddApiClientResponseInterceptor<Product, MyProductResponseInterceptor>();

// Specific to model + key
services.AddApiClientResponseInterceptor<Product, int, MyProductIntResponseInterceptor>();
```

Implement `IRepositoryResponseClientInterceptor`:

```csharp
public class MyResponseInterceptor : IRepositoryResponseClientInterceptor
{
    public async Task<HttpResponseMessage> CheckResponseAsync(
        HttpClient client,
        HttpResponseMessage response,
        Func<HttpClient, Task<HttpResponseMessage>> retry)
    {
        if (response.StatusCode == HttpStatusCode.Unauthorized)
        {
            // refresh token, then retry
            return await retry(client);
        }
        return response;
    }
}
```

## JWT / Bearer Authentication

The built-in bearer interceptor handles token injection and automatic 401 refresh.

### 1. Implement `ITokenManager`

```csharp
public class MyTokenManager : ITokenManager
{
    public Task<string?> GetTokenAsync()
    {
        // return current token
    }

    public Task<string?> RefreshTokenAsync()
    {
        // refresh and return new token
    }

    public async Task EnrichWithAuthorizationAsync(HttpClient client)
    {
        var token = await GetTokenAsync();
        client.DefaultRequestHeaders.Authorization =
            new AuthenticationHeaderValue("Bearer", token);
    }
}
```

### 2. Register the interceptor

```csharp
// Global — applies to all repository clients
services.AddDefaultAuthorizationInterceptorForApiHttpClient<MyTokenManager>();

// Specific to a model type
services.AddDefaultAuthorizationInterceptorForApiHttpClient<Product, MyTokenManager>();

// Specific to model + key type
services.AddDefaultAuthorizationInterceptorForApiHttpClient<Product, int, MyTokenManager>();
```

### `AuthenticatorSettings`

```csharp
services.AddDefaultAuthorizationInterceptorForApiHttpClient<MyTokenManager>(settings =>
{
    settings.Scopes = ["api.read", "api.write"];
    settings.ExceptionHandler = async (exception, serviceProvider) =>
    {
        // handle token acquisition errors
    };
});
```

The built-in interceptor automatically retries the request with a refreshed token on `401 Unauthorized`.

## Named Instances (multi-storage)

Match a named repository on the server side using `WithServerFactoryName`:

```csharp
builder.Services.AddRepository<Product, int>(repositoryBuilder =>
{
    repositoryBuilder.WithApiClient(apiBuilder =>
    {
        apiBuilder
            .WithHttpClient("https://api.example.com")
            .ApiBuilder
            .WithServerFactoryName("premium");
    });
});
```

This appends the factory name segment to all URLs:
`api/Product/premium/Get?key={key}`
